# AI 결과물 감리사 (AI Output Auditor)

경희대학교 미래직업 아이디어톤 발표용 웹사이트 프레젠테이션 (발표자: 란하, 2026. 6. 14.)

10개의 풀스크린 섹션으로 구성된 웹 기반 발표 자료입니다. 외부 CDN 의존이 전혀 없어
**발표장 인터넷 없이 완전히 오프라인으로 동작**합니다 (Pretendard 폰트 로컬 포함).

## 실행 방법

가장 간단하게는 `index.html`을 브라우저로 직접 열면 됩니다 (`file://` 동작 확인).

로컬 서버로 띄우려면:

```bash
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

## 발표 조작법

| 키 | 동작 |
|---|---|
| `→` `↓` `Space` `PageDown` | 다음 섹션 (발표 클리커 호환) |
| `←` `↑` `Shift+Space` `PageUp` | 이전 섹션 |
| `Home` / `End` | 첫 / 마지막 섹션 |
| `B` | 섹션 8 백업 자료 토글 (데모 실패 시 비상용) |

- 우측 하단에 `현재 섹션 / 10` 인디케이터 표시
- 섹션 진입 애니메이션은 1회만 실행 (발표 중 산만함 방지)
- 1920×1080(16:9) 빔프로젝터 기준 레이아웃

### 발표 운영 팁

- **애니메이션은 섹션 진입 순간 시작**되므로, 멘트를 먼저 시작하고 강조 타이밍에 → 키를 누르면 화면과 말이 맞아떨어집니다 (예: "78,000,000개가 많습니다" 직전에 섹션 3 진입 → 카운트업과 동시 강조)
- 섹션 10은 다짐 문구 → 도장이 회전하며 찍히는 애니메이션(2.8초) → 감사 인사(3.6초) 순으로 등장합니다. 진입 후 약 3초 침묵을 지키면 도장이 찍히는 순간이 발표의 클로징 절정이 됩니다 — 이 2초가 청중 기억에 가장 강하게 남는 장면이므로 멘트로 덮지 마세요
- 데모 실패 시: 데모 앱에서 자동으로 "오프라인 백업 결과 보기" 버튼이 나타나고(인터넷이 끊겨 있으면 샘플은 자동 전환), 발표 사이트에서는 `B` 키로 백업 표를 띄울 수 있습니다
- 데모 검토 단계 멘트: 결과가 뜨면 *"여기까지가 AI의 1차 분석입니다. 이제부터가 감리사의 일입니다"* 라고 말하며 2~3건을 직접 [인정] 클릭으로 확정 → "남은 항목 모두 인정" → 보고서 발행. 마무리 멘트: *"AI는 의심 구절을 찾고, 판단과 서명은 사람이 합니다 — 그게 이 직업입니다"*

## 데모 앱 (`demo/`) — 감리사 워크벤치

발표 섹션 8에서 시연하는 **감리사의 업무 도구(Auditor's Workbench)** 입니다.
"AI 서비스"가 아니라 **이 직업의 작업대**를 구현한 것 — AI는 의심 구절을 찾는 1차 분석 도구이고, 판정·서명은 사람만 할 수 있는 구조입니다. ("AI는 이 직업의 망치이고, 망치는 목수를 대체하지 않는다")
접수 → 리스크 검증 → 신뢰 점수 → 보고서 발행의 4단계가 발표 섹션 6의 프로세스와 1:1로 대응합니다.

- **실제 Claude API 연동** (`claude-opus-4-8`, 브라우저에서 직접 호출 — 서버 불필요)
- 가짜 판례가 섞인 샘플 의견서 원클릭 로드 + 텍스트 붙여넣기 + .txt/.md 업로드
- 환각 의심·편향·법적 리스크를 항목별 하이라이트, 신뢰 점수 링 애니메이션
- **감리사 검토 단계 (human-in-the-loop)**: AI 분석은 1차 제안일 뿐 — 감리사가 항목별로 [인정/제외]를 확정해야 보고서 발행 버튼이 활성화되고, 보고서에 "AI 분석 N건 → 감리사 인정 X · 제외 Y" 검토 기록이 남음. *"AI는 의심 구절을 찾고, 판단과 서명은 사람이 한다"는 직업 본질을 화면으로 증명하는 장치*
- 감리 보고서 발행: 서명란 "감리사 란하" + 감리 도장 포함, 브라우저 인쇄로 PDF 저장

### 데모 사용 준비 (발표 전 필수)

1. [console.anthropic.com](https://console.anthropic.com)에서 API 키 발급 (유료 — 소액 충전이면 충분, 1회 분석 약 $0.1 이하)
2. 데모 앱 우측 상단 **⚙ API 설정**에 키 입력 → 브라우저 localStorage에 저장됨 (발표용 노트북에서 미리 한 번 저장해두면 끝)
3. **주의: 데모 분석은 인터넷이 필요합니다.** 발표장 와이파이가 불안하면 휴대폰 핫스팟 준비 + 백업 영상 필수

## 예상 Q&A 대비

**Q. "사람 직업을 소개한다면서 구현품이 AI 서비스인 건 모순 아닌가?"**
A. 구현한 것은 AI 서비스가 아니라 **감리사의 작업대(워크벤치)** 입니다. 파일럿을 소개할 때 시뮬레이터를 보여주듯, 직업을 증명하는 가장 확실한 방법은 그 직업의 업무 환경을 보여주는 것입니다. 화면에서 보셨듯 AI가 할 수 있는 건 의심 구절 선별(1차 분석)까지이고, 항목별 인정/제외 판정과 실명 서명은 사람만 할 수 있게 잠겨 있습니다. AI는 이 직업의 망치이고, 망치는 목수를 대체하지 않습니다.

**Q. "감리사 본인이 틀리면? AI를 검증하는 사람은 누가 검증하나?"**
A. 감리사의 핵심 가치는 무오류가 아니라 **책임**입니다. 판사도 오류를 내지만 항소 제도가 있듯, 감리 보고서도 이의신청·재감리 절차를 전제합니다. AI와의 차이는 "틀렸을 때 책임지는 주체가 존재한다"는 것이고, 그래서 보고서에 실명 서명이 들어갑니다. 또한 판례 DB 대조 같은 기계적 검증은 자동화하고, 감리사는 해석·판정·책임에 집중합니다.

**Q. "AI가 몇 달마다 바뀌는데, 감리사의 검증 기준이 그 속도를 따라갈 수 있나?"**
A. 오히려 그 속도가 이 직업의 수요를 만듭니다. 감리는 "이 AI가 믿을만한가"를 한 번 도장 찍는 게 아니라, **특정 시점·특정 산출물의 한계를 문서화**하는 일입니다. 모델이 바뀔 때마다 기존 검증의 유효기간이 끝나므로 재감리 수요가 반복적으로 발생합니다 — 건물은 한 번 감리하면 끝이지만 AI는 갱신형 감리가 필요한 구조입니다. 감리 기준 자체는 파생 직업으로 제시한 '감리 표준 개발자'가 산업별로 업데이트합니다(섹션 9).

**Q. "자격증도 선발 기준도 없는데, 아무나 할 수 있는 일 아닌가? 지금 준비하는 사람은 뭘 해야 하나?"**
A. 모든 신뢰 직업이 같은 순서로 생겼습니다 — 공인회계사도 면허보다 '회계 감사를 하는 사람들'이 먼저 있었고, 제도가 그들을 자격증으로 공식화했습니다. 건축 감리 의무화 역시 대형 부실시공 사고를 겪은 뒤 법제화로 정착한 선례가 있습니다. 지금 단계(2026)는 기업 내부 검증 담당자·외부 컨설턴트로 존재하는 시기이고, 규제가 자격 요건을 명시하는 시점(2028 전후 예상)에 자격 체계가 따라옵니다. 지금 준비할 것은 세 가지입니다: ① 도메인 지식(법률·의료·금융 중 하나), ② AI의 실패 양식에 대한 이해(환각·편향이 어떻게 생기는지), ③ 검증 도구 활용 능력 — 오늘 데모로 보여드린 것이 바로 ③의 도구입니다.

**Q. "규제는 생겼지만 실제 채용 수요가 있나? 2026년 지금 필요한 직업인가?"**
A. 전기안전 기사도 전기설비 규제 시행 이후에 직업으로 정착했듯, 규제가 수요를 만듭니다. 한국 AI기본법(2026. 1. 시행)은 고영향 AI에 위험관리 의무를 부과했고, 기업들은 지금 내부 검증 체계를 급히 만드는 단계입니다. 초기 시장은 법률 의견서·의료 보고서·금융 문서의 외부 검수 서비스 형태로 열리고, EU 고위험 AI 전면 규제(2027. 12.)가 본격적인 수요 기점이 될 것으로 봅니다.
